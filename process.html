Hur tänkte jag här...?

Den här uppgiften tycker jag har varit givande för mig. Jag har fått klurat en hel del. Bra att ni lagt in att vi ska tänka på stylingen för @print regeln, då jag aldrig använt mig av den förut.

Jag har valt att använda mig av CSS Grid (2017) för mig layout. För jag anser att det är ett lättare sätt att jobba med när man vill göra en layout. Dels för att det blir mindre kod att skriva samt att det är lättare att överskåda och förstå syntaxen. Sedan tycker jag det blir väldigt krångligt med float när man har många element som är floatade. Jag tycker det är svårt att debugga.
Men jag tycker det ändå är lättare att förstå CSS Grid (2017). Jag antar att det kanske är en smaksak till en början. Sedan så tänkte jag på om jag skulle köra på floats. I och med att det är en så enkel layout att göra float den här uppgiften.

Man kan säga att det nästan blev mer kod när jag använde mig av CSS Grid (2017) på ett vis. Exemplet nedan är hur jag skulle använda mig av ser ut floats.

Exempel: 

	
	.element {
		float: left
	}

	.clearfix:after {
		content:" ";
		display:table;
		clear:both;
	}

Medans det blir med kod med CSS Grids (2017). Men det är på ett ju också för att man kan göra så mycket mera med den egenskapen. Som en fallback så har jag först använt mig av @supports (2017) regeln. Den tycker jag är ett måste om jag vill att så många webbläsare kan ta del av min layout eller vad det nu är jag ska åstakomma.

När man använder sig av @supports (2017) så kommer din webbläsare automatiskt veta om den har någon support för @supports (2017) regeln. Gör den inte det så kommer den att läsa på nästa rad i CSS filen och agera utefter vilken kod som står där.

Exempel: 

	
	.features }
		@supports (display: grid) {
			display: grid
			grid-gap: 1rem
			grid-template-columns: repeat(3, 1fr)
			grid-template-rows: 1fr
		}
		display: flex
		flex-direction: column
		}
